# 08_知识蒸馏

本章对应 `minimind_src/trainer/train_distillation.py`，实现 MiniMind 的知识蒸馏（Knowledge Distillation）。

该脚本的核心训练目标是：

1. **CE（真实标签）项**：student 去拟合 ground-truth（来自 SFT 的 `labels`）
2. **Distill（蒸馏 KL）项**：student 去拟合 teacher 的“软目标分布”（温度缩放后）
3. **加权合成**：
   - 总损失 `loss = (alpha * ce_loss + (1-alpha) * distill_loss) / accumulation_steps`

其中 `alpha` 与 `temperature` 决定两者权重与软化程度。

## 2. `distillation_loss`（逐行+张量维度）

```python
24: def distillation_loss(student_logits, teacher_logits, temperature=1.0, reduction='batchmean'):
25:     with torch.no_grad():
26:         teacher_probs = F.softmax(teacher_logits / temperature, dim=-1).detach()
27:
28:     student_log_probs = F.log_softmax(student_logits / temperature, dim=-1)
29:
30:     kl = F.kl_div(
31:         student_log_probs,
32:         teacher_probs,
33:         reduction=reduction
34:     )
35:     return (temperature ** 2) * kl
```

### 2.1 蒸馏张量形状约定

在调用处（见后面 `train_epoch`），student/teacher 传入的是经过 mask 过滤后的“有效 token 行集合”，因此通常形状为：

- `student_logits: [N, V]`
- `teacher_logits: [N, V]`
- `teacher_probs: [N, V]`
- `student_log_probs: [N, V]`

其中：

- `N` 是有效 token 数（assistant 区间 token 的数量），不是 batch size。

### 2.2 逐行解释

- 第 26 行：`teacher_probs = softmax(teacher_logits/temperature)` 得到 teacher 的软分布
  - `/temperature` 会把分布“软化”（temperature 越大越平滑）
  - `.detach()` 保证 teacher 不参与反传
- 第 28 行：student 侧取 log-softmax
- 第 30-34 行：`F.kl_div(student_log_probs, teacher_probs)` 计算 KL
  - reduction 默认 `batchmean`，最终是标量（或张量但这里通常标量）
- 第 35 行：乘 `(temperature**2)` 是蒸馏的标准修正，保证梯度尺度随温度合适调整。

## 3.3 loss_mask：从 labels 的 ignore_index 构造有效 token mask（逐行+维度）

```python
50: loss_mask = (labels[..., 1:] != -100).float()
```

维度追踪：

- `labels[..., 1:]`：`[B, T] -> [B, T-1]`
- `!= -100` 得到 bool mask `[B, T-1]`
- `.float()` -> `loss_mask: [B, T-1]`

语义：

- 只在 assistant 输出区间（labels 非 -100 的位置）计算 CE 与蒸馏 KL。

## 3.5 student 前向：取 `logits[..., :-1, :]`

```python
57: res = model(input_ids)
58: student_logits = res.logits[..., :-1, :].contiguous()
```

维度：

- `res.logits: [B, T, V]`（因为模型输出与输入序列对齐）
- `[..., :-1, :]`：`[B, T-1, V]`
- `student_logits: [B, T-1, V]`

此切片与 CE 的 shift_labels 是同一个时间对齐约定：预测位置 t 对应真实标签 t+1。

## 3.6 teacher 前向（无梯度）与 vocab 对齐

```python
61: with torch.no_grad():
63:     teacher_logits = teacher_model(input_ids).logits[..., :-1, :].contiguous()
64:     vocab_size_student = student_logits.size(-1)
65:     teacher_logits = teacher_logits[..., :vocab_size_student]
```

维度：

- `teacher_logits: [B, T-1, V_teacher]`
- 截断到 student vocab：`teacher_logits[..., :V_student] -> [B, T-1, V_student]`

动机：

- 若 teacher 与 student tokenizer/词表大小可能不同，则通过截断保证二者维度可一致计算 KL。

## 3.7 计算 CE loss（逐行+mask 聚合）

```python
68: shift_labels = labels[..., 1:].contiguous()
70: loss_mask_flat = loss_mask.view(-1)
71: ce_loss = F.cross_entropy(
72:     student_logits.view(-1, student_logits.size(-1)),
73:     shift_labels.view(-1),
74:     ignore_index=-100,
75:     reduction='none'
76: )
77: ce_loss_raw = torch.sum(ce_loss * loss_mask_flat) / (loss_mask_flat.sum() + 1e-8)
```

维度追踪：

- `shift_labels: [B, T-1]`
- `loss_mask_flat: loss_mask.view(-1) -> [B*(T-1)]`
- `student_logits.view(-1, V): [B*(T-1), V]`
- `F.cross_entropy(... reduction='none')` 返回每个 token 的损失：
  - `ce_loss: [B*(T-1)]`

聚合：

- `ce_loss * loss_mask_flat`：只保留有效 token 的损失（mask 为 0 的位置乘掉）
- `sum(...) / loss_mask_flat.sum()`：得到有效 token 的平均 CE。

### 3.7.1 MoE 情况下 aux_loss 的加入方式

```python
78: if lm_config_student.use_moe: ce_loss = ce_loss_raw + res.aux_loss
79: else: ce_loss = ce_loss_raw
```

- `res.aux_loss` 是一个标量，来自 student 模型内部 MoE router 正则项
- 因此当 student.use_moe=True：
  - `ce_loss` 不再只是 CE 标量，而变成 `CE + aux_loss`

## 3.8 计算蒸馏 KL loss（逐行+mask 选择 token 行）

```python
82: if teacher_model is not None:
83:     distill_loss = distillation_loss(
84:         student_logits.view(-1, student_logits.size(-1))[loss_mask_flat == 1],
85:         teacher_logits.view(-1, teacher_logits.size(-1))[loss_mask_flat == 1],
86:         temperature=temperature
87:     )
89: else:
90:     distill_loss = torch.tensor(0.0, device=args.device)
```

维度追踪：

- `student_logits.view(-1, V): [B*(T-1), V]`
- `loss_mask_flat==1` 是 bool 索引，选出有效行：
  - 令有效 token 数为 `N`：
  - 选后 `student: [N, V]`
  - 选后 `teacher: [N, V]`

- `distillation_loss` 输出标量（KL + temp^2）

## 3.9 合成总损失并反传

```python
91: loss = (alpha * ce_loss + (1 - alpha) * distill_loss) / args.accumulation_steps
94: scaler.scale(loss).backward()
```

标量合成：

- `ce_loss: scalar`
- `distill_loss: scalar`
- `loss: scalar`
- 再除以 accumulation_steps 保持梯度尺度一致

## 4. `__main__`：student/teacher 可不同配置（逐行）

关键区别在构建两个 config：

```python
185: lm_config_student = MiniMindConfig(... student_use_moe=...)
186: lm_config_teacher = MiniMindConfig(... teacher_use_moe=...)
```

因此蒸馏允许：

- student/teacher 是否 MoE 不同（CE aux_loss 只在 student.use_moe 时加入；teacher 的 aux_loss 不参与，因为 teacher 不参与 backward，且脚本直接读取其 logits）

## 5. 本章小结（复核要点）

1. CE 与蒸馏 KL 都只在 `labels[...,1:] != -100` 的 token 上计算
   - 通过 `loss_mask` -> `loss_mask_flat` 实现 token 行过滤
2. `student_logits` / `teacher_logits` 切片后维度为：
   - `[B, T, V] -> [B, T-1, V] -> view(-1,V) -> [B*(T-1),V]`
3. 蒸馏 KL 传入的是有效 token 行集合：
   - `[B*(T-1),V]` 通过 `loss_mask_flat==1` 选为 `[N,V]`
4. `temperature**2` 是标准 KD 修正项
5. student MoE 时：`ce_loss = CE + res.aux_loss`，并被 `alpha` 加权进入总损失
