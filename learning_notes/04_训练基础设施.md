# 04_训练基础设施（分布式/检查点/工程优化）

本章对应 `minimind_src/trainer/trainer_utils.py`。它把“训练脚手架”中最容易看漏但会强烈影响训练行为的部分集中到一个文件里：

- 分布式初始化（DDP）与随机种子
- checkpoint 保存/恢复（含 world_size/step 对齐）
- 学习率余弦调度
- “跳过前 N 个 batch”的断点续训机制（`SkipBatchSampler`）
- 奖励模型（Reward Model / sequence-scoring）输入拼接与输出裁剪
- 模型参数量统计（尤其 MoE 部分的 active/routed 参数估计）

## 2. `get_model_params`：MoE 参数量估计

```python
18: def get_model_params(model, config):
19:     total = sum(p.numel() for p in model.parameters()) / 1e6
20:     n_routed = getattr(config, 'n_routed_experts', getattr(config, 'num_experts', 0))
21:     n_active = getattr(config, 'num_experts_per_tok', 0)
22:     n_shared = getattr(config, 'n_shared_experts', 0)
23:     expert = sum(p.numel() for n, p in model.named_parameters() if 'mlp.experts.0.' in n) / 1e6
24:     shared_expert = sum(p.numel() for n, p in model.named_parameters() if 'mlp.shared_experts.0.' in n) / 1e6
25:     base = total - (expert * n_routed) - (shared_expert * n_shared)
26:     active = base + (expert * n_active) + (shared_expert * n_shared)
27:     if active < total: Logger(f'Model Params: {total:.2f}M-A{active:.2f}M')
28:     else: Logger(f'Model Params: {total:.2f}M')
```

关键点：

- `numel()` 返回参数张量元素数，是标量计数，不涉及维度变换。
- `expert` 和 `shared_expert` 是用 name pattern 估计单个 expert 的参数量：
  - 这里假设 MoE 中子模块命名包含：
    - `mlp.experts.0.`（第 0 个 routed expert）
    - `mlp.shared_experts.0.`（共享 expert）
- `base` 是把 routed/shared expert 总参数从 `total` 里扣掉后的“骨干部分”。
- `active` 是推理/训练时“每 token 实际激活”的 expert 数对应的参数量估计：
  - `active = base + expert * n_active + shared_expert * n_shared`

动机：

- MoE 的计算量与激活参数相关，因此常同时给出 `total` 和“active”参数量估计，让你理解稀疏路由下的有效规模。

## 3. `is_main_process` 与 `Logger`：只在 rank0 打印

```python
31: def is_main_process():
32:     return not dist.is_initialized() or dist.get_rank() == 0

35: def Logger(content):
36:     if is_main_process():
37:         print(content)
```

语义：

- 非 DDP：`dist.is_initialized()==False`，所以 rank0 概念等价为当前进程。
- DDP：仅在 `rank==0` 打印，避免多卡刷屏。

## 4. `get_lr`：余弦余量策略

```python
40: def get_lr(current_step, total_steps, lr):
41:     return lr*(0.1 + 0.45*(1 + math.cos(math.pi * current_step / total_steps)))
```

动机：

- `cos` 型调度：在训练开始与中后期会有不同衰减速度。
- 该函数返回的是一个标量学习率。
- 注意：它是“手动设置 lr”的函数，训练脚本中会循环里对 optimizer 的 `param_group['lr']=lr`。

## 5. `init_distributed_mode`：DDP 初始化

```python
44: def init_distributed_mode():
45:     if int(os.environ.get("RANK", -1)) == -1:
46:         return 0  # 非DDP模式

48:     dist.init_process_group(backend="nccl")
49:     local_rank = int(os.environ["LOCAL_RANK"])
50:     torch.cuda.set_device(local_rank)
51:     return local_rank
```

语义：

- 看环境变量 `RANK`：
  - `RANK` 不存在或为 -1：认为不是 DDP，返回 0。
  - 否则：
    - 初始化进程组：`backend="nccl"`（GPU 通常用 NCCL）。
    - 从 `LOCAL_RANK` 取本机卡号，并 `set_device`，保证每个进程只用自己的 GPU。

风险/假设：

- 该实现强依赖外部启动方式（例如 torchrun）正确设置 `RANK/LOCAL_RANK`。

## 7. `lm_checkpoint`：保存/恢复全训练状态

这是最关键的工程函数之一。

### 7.1 保存模式（`model is not None`）

```python
63: def lm_checkpoint(..., model=None, optimizer=None, epoch=0, step=0, wandb=None, save_dir='../checkpoints', **kwargs):
64:     os.makedirs(save_dir, exist_ok=True)
65:     moe_path = '_moe' if lm_config.use_moe else ''
66:     ckp_path = f'{save_dir}/{weight}_{lm_config.hidden_size}{moe_path}.pth'
67:     resume_path = f'{save_dir}/{weight}_{lm_config.hidden_size}{moe_path}_resume.pth'

69:     if model is not None:
70:         raw_model = model.module if isinstance(model, DistributedDataParallel) else model
71:         raw_model = getattr(raw_model, '_orig_mod', raw_model)
72:         state_dict = raw_model.state_dict()
73:         state_dict = {k: v.half().cpu() for k, v in state_dict.items()}
74:         ckp_tmp = ckp_path + '.tmp'
75:         torch.save(state_dict, ckp_tmp)
76:         os.replace(ckp_tmp, ckp_path)
```

关键点：

- `raw_model` 解包 DDP：从 `model.module` 取底层模型。
- `_orig_mod` 解包 torch.compile 产生的 wrapper（避免保存到 wrapper 名字不匹配）。
- `state_dict` 保存为 fp16（`v.half()`）并移动到 CPU：
  - 减少 GPU 显存峰值与 checkpoint 文件体积。
- 通过写临时文件 `ckp_tmp` 并 `os.replace` 原子替换，避免保存过程中中断导致文件损坏。

下一段保存 resume：

```python
77:         wandb_id = None
78:         if wandb:
79:             if hasattr(wandb, 'get_run'):
80:                 run = wandb.get_run()
81:                 wandb_id = getattr(run, 'id', None) if run else None
82:             else:
83:                 wandb_id = getattr(wandb, 'id', None)

85:         resume_data = {
86:             'model': state_dict,
87:             'optimizer': optimizer.state_dict(),
88:             'epoch': epoch,
89:             'step': step,
90:             'world_size': dist.get_world_size() if dist.is_initialized() else 1,
91:             'wandb_id': wandb_id
92:         }
```

字段语义：

- `epoch/step/world_size` 用于恢复时对齐学习率/step（尤其当你用不同 GPU 数启动续训时）。
- `optimizer.state_dict()`：包含动量/二阶矩等，确保 AdamW 能从同样的优化状态继续。
- `model` 保存的是前面已经半精度 CPU 的 `state_dict`（和 `ckp_path` 同源）。
- `wandb_id`：若要 resume wandb，会用到它。

`kwargs` 允许额外保存 scheduler、scaler 等：

```python
93:         for key, value in kwargs.items():
94:             if value is not None:
95:                 if hasattr(value, 'state_dict'):
96:                     raw_value = value.module if isinstance(value, DistributedDataParallel) else value
97:                     raw_value = getattr(raw_value, '_orig_mod', raw_value)
98:                     resume_data[key] = raw_value.state_dict()
99:                 else:
100:                    resume_data[key] = value
101:
102:        resume_tmp = resume_path + '.tmp'
103:        torch.save(resume_data, resume_tmp)
104:        os.replace(resume_tmp, resume_path)
105:        del state_dict, resume_data
106:        torch.cuda.empty_cache()
```

注意：

- `hasattr(value,'state_dict')`：只要对象能提供 state_dict，就保存它。
- 这允许训练脚本传入 `scaler=GradScaler`、`scheduler=CosineAnnealingLR`、以及 `critic_model/actor_model` 等（只要它们具有 state_dict）。

### 7.2 加载模式（`model is None`）

```python
107:    else:  # 加载模式
108:        if os.path.exists(resume_path):
109:            ckp_data = torch.load(resume_path, map_location='cpu')
110:            saved_ws = ckp_data.get('world_size', 1)
111:            current_ws = dist.get_world_size() if dist.is_initialized() else 1
112:            if saved_ws != current_ws:
113:                ckp_data['step'] = ckp_data['step'] * saved_ws // current_ws
114:                Logger(f'GPU数量变化({saved_ws}→{current_ws})，step已自动转换为{ckp_data["step"]}')
115:            return ckp_data
116:        return None
```

世界大小对齐细节：

- 若 checkpoint 保存时 GPU 数 `saved_ws` 与当前 `current_ws` 不同：
  - 以整数比例 `step' = step * saved_ws // current_ws` 进行缩放。
  - 目的：在“每 step 处理的样本数”变化时，尽量让学习率/训练进度相近。

## 9. `SkipBatchSampler`：跳过前 N 个 batch

```python
134: class SkipBatchSampler(Sampler):
135:     def __init__(self, sampler, batch_size, skip_batches=0):
136:         self.sampler = sampler
137:         self.batch_size = batch_size
138:         self.skip_batches = skip_batches
```

迭代器实现：

```python
140:     def __iter__(self):
141:         batch = []
142:         skipped = 0
143:         for idx in self.sampler:
144:             batch.append(idx)
145:             if len(batch) == self.batch_size:
146:                 if skipped < self.skip_batches:
147:                     skipped += 1
148:                     batch = []
149:                     continue
150:                 yield batch
151:                 batch = []
152:         if len(batch) > 0 and skipped >= self.skip_batches:
153:             yield batch
```

语义：

- 传入一个“基础采样器”（可能是 DistributedSampler 或 randperm indices）。
- 按 `batch_size` 聚合为 batch。
- 对前 `skip_batches` 个 batch 不产出，从而实现“断点续训从 step 开始继续”。

长度估计：

```python
155:     def __len__(self):
156:         total_batches = (len(self.sampler) + self.batch_size - 1) // self.batch_size
157:         return max(0, total_batches - self.skip_batches)
```

## 10. `LMForRewardModel`：奖励模型评分

Reward 模型不是在本文件里训练的，而是被 PPO/GRPO/AgentRL 的 reward 计算调用。

```python
160: class LMForRewardModel:
161:     def __init__(self, model_path, device="cuda", dtype=torch.float16):
162:         self.tokenizer = AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)
163:         self.model = AutoModel.from_pretrained(model_path, torch_dtype=dtype, trust_remote_code=True)
164:         self.model = self.model.to(device).eval()
165:         self.device = device
```

评分函数：

```python
167:     @torch.no_grad()
168:     def get_score(self, messages, response):
169:         history_text = "\n".join([f"{m['role']}: {m['content']}" for m in messages[:-1]])
170:         last_query = messages[-1]['content'] if messages else ""
171:         message_context = f"{history_text}\n以上是对话历史。我的新问题是：\n{last_query}" if history_text else last_query
172:         eval_messages = [
173:             {"role": "user", "content": message_context},
174:             {"role": "assistant", "content": response}
175:         ]
176:         score = self.model.get_score(self.tokenizer, eval_messages)
177:         return max(min(score, 3.0), -3.0)
```

关键点：

- `messages` 传入的是一个对话列表：最后一条通常是 user query，倒数第二条及之前是 history。
- 该 RewardModel 需要 `model.get_score(tokenizer, eval_messages)` 方法（不属于标准 HF 接口），因此这份代码假定 `trust_remote_code=True` 加载的模型实现了 `get_score`。
- 输出裁剪到 `[-3, 3]`：
  - PPO/GRPO/AgentRL 的优势计算对奖励幅度敏感，裁剪能避免极端值导致训练不稳定。

维度说明：

- `score` 在这里是 Python 数值/标量（未在代码中显式做 tensor 维度变换）。

## 11. 本章小结

本章中：

1. DDP 是否启动完全由 `RANK` 环境变量决定，`LOCAL_RANK` 决定 GPU 绑定。
2. `lm_checkpoint` 同时保存：
   - 半精度 CPU 的 `model.state_dict`（用于推理/权重加载）
   - 以及可恢复训练状态的 `resume_data`（model/optimizer/epoch/step/world_size/可能的 scheduler/scaler）
3. 恢复时如果 GPU 数变化，会对 `step` 按 `saved_ws/current_ws` 做整数缩放。
4. `SkipBatchSampler` 通过跳过前 `skip_batches` 个 batch 实现断点续训的“数据位置对齐”。
5. RewardModel 的输入拼接是把 history 串成一个 context，再调用模型的非标准 `get_score`，最后对 score 做 `[-3,3]` 裁剪。

下一章 `05_预训练与全量 SFT 训练循环` 会逐行解析 `train_pretrain.py` 和 `train_full_sft.py`：包括如何使用本章的这些脚手架（`init_distributed_mode/setup_seed/lm_checkpoint/SkipBatchSampler/get_lr`），并把 `res.loss + res.aux_loss` 合成总 loss。
